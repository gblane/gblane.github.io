Simulation of Apparent Measurements

Simulation of Apparent Measurements

In general we simulate a given measurement by generating forward data with a forward model and then analyzing that data. The analysis process may be considered an inverse model which is in turn based on a forward model itself. Therefore these simulations are interesting to the extent that the forward model used to generate data and inverse model do not match.

Simulation of Measurements of Dynamic Changes

When analyzing data for Δμa the analysis methods are those found in Appendix [app:dmuaMeas]. These methods are based on retrieving Δμa as if the change occurred everywhere in a semi-infinite homogeneous medium. Therefore if the change is localized, what is measured will not be the same as the actual Δμa. This relationship depends on analysis method and perturbation location which is the meaning of 𝒮 maps (Appendix [app:senMaps]). Therefore, these simulations are done using 𝒮 as the forward model and Appendix [app:dmuaMeas] as the inverse.

Assume a medium is voxalized into \(N_{vox}\) voxels without overlap. Then using the methods in Appendix [app:senMaps], a \(N_{vox}\times 1\) vector of 𝒮 (\(\harp{\mathcal{S}}\)) can be created for whichever datatype one wishes to simulate. Now for the same voxels we can simulate a actual Δμa \(N_{vox}\times 1\) vector (\(\acrshort{dmuaVec}_{,actual}\)). Rearranging Equation [equ:sen_dmua] and summing over all voxels we can calculate the apparent measured Δμa (\(\Delta\mu_a_{,apparent}\)): \[\acrshort{dmua}_{,apparent}=\harp{\acrshort{sen}}^T \acrshort{dmuaVec}_{,actual}\] It is emphasized that this method only works if the 𝒮 values correspond to voxels which do not overlap, and of course the order of the voxels in the two vectors must be the same.

"

Now lets look at this implementation in MATLAB. The code in Listing [lst:senMaps] can be used to generate 𝒮 maps. To do the type of simulation we wish we must not have overlap. This is achieved by setting the "pert.partSz" input to "[1,1,1]". Now the function will retrieve the 𝒮 for each individual voxel without using convolution. Lets utilize a specific example to demonstrate the next steps. Say we want to simulate a SD set with ρ of 35 mm, the default medium optical properties (Listing [lst:senMaps]), and a actual Δμ⃗a ("dmua_actual") of 0.001  mm at all voxels with \(z\geq\)10 mm. The following code will retrieve the apparent measured Δμa with SD I ("dmua_apparent") using \(\harp{\acrshort{sen}}\) ("Svec") from Listing [lst:senMaps]:

armt.rSrc=[-35/2, 0, 0]; %mm
armt.rDet=[35/2, 0, 0]; %mm
armt.SDprs=[1, 1];

pert.partSz=[1, 1, 1];
%pert.dmua not used in this example
pert.dmua=NaN;

[SD, ~, ~, params]=...
    makeSenMaps(armt, pert);
Svec=SD.S_In(:);

dmua_actual=...
    (params.geo.Z(:)>=10)*...
    0.001; %1/mm

dmua_apparent=Svec'*dmua_actual;

This method can be expanded to different actual Δμa in different voxels allowing one to simulate measurements from regions with different Δμa dynamics. "

Using the above equation and code we can simulate a apparent measured Δμa given localized actual Δμa. However, often we to simulate ΔO and ΔD. This turns out to be simple, the forward model for Δμa from Appendix [app:forMod] can be used to find actual Δμa at every voxel from the actual ΔO and ΔD at each voxel. Then the above simulation methods find apparent Δμa which can be converted to apparent ΔO and ΔD with Appendix [app:recChrom]. If this is done, the simulation needs to be run at two or more λs.

Furthermore, if one wishes to simulate Õ and D̃ the same methods can also be applied. The forward models to retrieve Δμa from Appendix [app:forMod] will accept complex numbers which represent Õ and D̃ and will result the μa phasors. Then the 𝒮 methods will also work with complex numbers to give the apparent μa phasor, and finally these complex μa phasors can be input into the methods in Appendix [app:recChrom] to result in apparent Õ and D̃.

Therefore, using 𝒮 maps one can simulate various types of Δμas (including phasors) and receive the apparent measured Δμas. However, one should not that what is simulated is dependent in the absolute μa and μ′s, which is also assumed to be homogeneous in the 𝒮 expressions used here. Additionally, the 𝒮 theory assumes Δμa is small compared to μa. This simplification is not realistic and should be noted using 𝒮 simulations. But certain types of simulations are still valuable, such as comparing what different data-types measure from the same medium with simulated Δμa.

Simulation of Measurements of Absolute Optical Properties

As stated above, the simulations in this appendix are primarily concerned with testing mismatched forward and inverse models. In the case of the measurement of absolute optical properties, the inverse model is the one in Appendix [app:absOptPropMeas] which is based on the semi-infinite homogeneous medium. Most of the interest in testing this inversion comes from generating forward data from heterogeneous media. This was done extensively in Part [prt:DSspect].

Our inverse model for absolute optical properties (Appendix [app:absOptPropMeas]) takes measurements of R̃ at two ρ as input. Therefore any forward model which calculates R̃ as a function of ρ may be used to generate forward data for the inverse model. The first such forward model is the two-layer forward model in Appendix [app:forMod] which gives R̃ from a medium with two layers having different μa and μ′s. Another forward model that may be used is Monte Carlo which can simulate even more complex distributions of μa and μ′s to generate R̃ which also can be inverted with the methods in Appendix [app:absOptPropMeas].

In practice this is quite simple. Any forward model you like is used to generate R̃ at at least two ρs. Then this R̃ is input into the methods in Appendix [app:absOptPropMeas] to result in the effective homogeneous μa and μ′s. The result from this computational experiment is called effective homogeneous because the inverse model has found a homogeneous medium with μa and μ′s that would result in a similar or even the same behavior in R̃ as the more complex forward model. Studying how effective homogeneous optical properties are related to actual heterogeneous optical properties is complex, in part because it is likely that different heterogeneous optical properties may result in the same effective homogeneous optical properties.
